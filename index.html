Le fonctionnement général :

Un développeur du groupe de développement, crée un dépôt (repository), qui contiendra le code source du projet. Les autres développeurs récupéreront le code pour pouvoir travailler sur celui-ci. Avant de commencer à travailler ils créeront différentes branches, ainsi le travail ne sera pas modifié à la source. Ils peuvent récupérer l’avancement du travail en tirant « pull » le projet de GitHub. A tout moment ils peuvent créer un « add » c’est à dire ajouter un ou plusieurs fichier(s)/dossier(s), dans la base des objets avant d’être d’être impliqué « commit » avant l’envoye sur github. Ils enverront sur GitHub « push » en poussant le projet sur leur branche. Ils peuvent à tout moment travailler sur le projet d’une autre branche ou même combiner « merge » leur travails sur la branche principale.
